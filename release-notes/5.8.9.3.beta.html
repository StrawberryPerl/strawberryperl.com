Strawberry Perl Release Notes

Version 5.8.9.3 Beta 2, released 21 Sep 2009

Check out what is new, what known issues there are, and frequently asked questions about this version of Strawberry Perl. As always, you're encouraged to tell us what you think.

What's new in Strawberry Perl 5.8.9.3 Beta 2: Collapse

This release was built with an updated version of the builder modules than were used for the July 2009 versions (5.10.0.6/5.8.9.2) of Strawberry Perl. In particular, there is now a link to these release notes installed when Strawberry Perl is installed, and non-core modules Strawberry Perl installs are now in /strawberry/perl/vendor/lib, instead of /strawberry/perl/site/lib.

All modules installed with Strawberry are updated.

DBD::Pg, Imager, GD, Net::SMTP::TLS, Crypt::SSLeay (so LWP can now handle https: sites), and Module::Signature are included.

Current developer releases of BioPerl are now easily installable. (Just don't add SOAP::Lite as an optional dependency.)

NOTE: This is considered a beta version of "Strawberry Perl 5.8.9.3", so uninstallation of this version will not be automatic when that version of Strawberry Perl is released.


Known issues: Collapse

  • Net::SSH::Perl is included (it installs) but does not work. It will be removed from the final version of Strawberry Perl if it is not fixed. RT#49820
  • When uninstalling Strawberry Perl, there is no feedback when searching for files to delete, and it isn't as fast as it could be. To work around this problem for now, you'll want to delete the strawberry\cpan, strawberry\cpanplus, and strawberry\perl\site directories before uninstalling or upgrading.RT#50028
  • The CPAN client closes too quickly when the lock file exists.
  • Tk will not install. RT#49942
  • This is considered a beta version of "Strawberry Perl 5.8.9.3", so uninstallation of this version will not be automatic when that version of Strawberry Perl is released. This release is recommended for testing purposes only.
