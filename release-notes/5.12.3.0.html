Strawberry Perl Release Notes

Version 5.12.3.0, built 15 May 2011, to be released 19 May 2011

Check out what is new, what known issues there are, and frequently asked questions about this version of Strawberry Perl. As always, you're encouraged to tell us what you think.

What's new in Strawberry Perl 5.12.3.0: Collapse

A new version of Perl: 5.12.3.

Also includes updated versions of all modules included in 5.12.2.0.

The .msi package has now been signed, for additional assurance.

The bug in the .msi (RT#62853) that made it crash during relocation has been fixed (a few files were missing from the 5.12.1.0 and 5.12.2.0 .msi's. The build process now makes sure that all the files that will be in the .zip will also be in the .msi.)

The Math::BigInt::GMP and Math::GMP modules (and modules that have one of them as prerequisites) are now usable on older machines.


Known issues: Collapse

  • The 64-bit version does not contain all of the modules that the 32-bit version does. Strawberry Perl is waiting on updates to those modules, as the policy of Strawberry Perl maintainers is not to patch any CPAN modules themselves, but to report required patches to the module author for inclusion in the next version.
    Known issues are Math::Pari (RT#52146), Win32::API (RT#42685), Crypt::IDEA (RT#55158), Crypt::Blowfish (RT#55173), and Crypt::SSLeay (RT#52408).

    There are other modules that are not included with Strawberry Perl that will also have problems with compiling in the 64-bit version that will compile in the 32-bit version. Please report any such problems to the module's author(s), and let us know as well.
  • PAR::Packer will work with the 32-bit version of Strawberry Perl. A compiled resource file for the 64-bit version will have to be created before PAR::Packer can be used with it. Watch RT#57273 for more information.
  • DBD::MySQL 4.019 will not build on any version of Strawberry Perl. This has been reported to the author. (4.018 will build, and is included with Strawberry Perl.)
