Strawberry Perl Release Notes

Version 5.12.0.1, released 06 May 2010

Check out what is new, what known issues there are, and frequently asked questions about this version of Strawberry Perl. As always, you're encouraged to tell us what you think.

What's new in Strawberry Perl 5.12.0.1: Collapse

  • New version of Perl: 5.12.0.
  • Updated all included dynamic link libraries with ones that will not conflict with anything else on the system.
  • The MPFR and MPC mathematics libraries, and the modules that use them (Math::MPFR and Math::MPC) are now included.
  • Added File::Slurp, the .xs compression library, a version-checking script, and a local::lib helper script.
  • Added the ability to install Strawberry Perl in any directory without spaces.
  • PPM now does not try to install .html files. (If you want POD in your browser, we recommend installing Perldoc::Server.)
  • XML::Simple has now been added. (New versions of Win32::Exe want it, so it's included.)

    • Known issues: Collapse

    • If $ENV{'SystemRoot'} is undefined (by a local %ENV or local $ENV{'SystemRoot'}, perl crashes.
    • CPAN will occasionally fail to move the directory that a distribution is temporarily unpacked to. (RT#55924)
    • The 64-bit version does not contain all of the modules that the 32-bit version does. Strawberry Perl is waiting on updates to those modules, as the policy of Strawberry Perl maintainers is not to patch any CPAN modules themselves, but to report required patches to the module author for inclusion in the next version.
      Known issues are Math::Pari (RT#52146), Win32::API (RT#42685), Crypt::IDEA (RT#55158), Crypt::Blowfish (RT#55173), and Crypt::SSLeay (RT#52408).

      There are other modules that are not included with Strawberry Perl that will also have problems with compiling in the 64-bit version that will compile in the 32-bit version. Please report any such problems to the module's author(s), and let us know as well. Devel::Caller is a known problem as of this writing, and Tk requires installation of a development version. ("install SREZIC/Tk-804.028_503.tar.gz" at a CPAN prompt, and if there are any other problems, ask on IRC.)
    • PPM has been removed from the 64-bit version, as there are no repositories that this version of PPM can use. (RT#57197)
    • PAR::Packer will work with the 32-bit version of Strawberry Perl, but please note that if you're creating an executable with PAR::Packer's pp script, you'll need to add -l C:\strawberry\perl\bin\libgcc_s_sjlj-1.dll to the pp command line (or @file) that you create the .exe with. Because of this, PAR::Packer 1.004 fails one test. This failure can be ignored (by installing with "notest install PAR::Packer" at the CPAN prompt), and has been reported to the module authors. (A compiled resource file for the 64-bit version will have to be created before PAR::Packer can be used with it. Watch RT#57273 for more information.)
